A Structure for Pythonic Thought

Disregard At Your Own Peril

One of the most annoying issues facing the initial implementation of a Python project is starting with a good initial project structure. A good project structure that enables ease of development, and provides for maintaining an adequate level of quality in the code under development.

A basic project structure should provide the facilities for:

Project Structure Facility Requirements
- Source Control Configuration
- Dev Environment Configuration
- Package Control and Configuration
- Test Configuration and Execution
- Documentation Generation

In the initial conception of thought into code, it is quite easy to forgo the need for any of the basics that make for an adequate project structure. It would be erroneous to blame a lack of adequate tools or standards. All of the popular programming languages have facilities for project infrastructures. The culprit in most cases are the desire to place thoughts into code, and the sheer number of alternative tools and schools of thought. To add to the complexity, it is often the case that for service and cloud oriented projects there will eventually come the need to implement CI and Ops requirements that at the initial stages of a project cannot be adequately predicted.

Sometimes Fish Out of Water are Evolving

Presented in this contribution is a minimal Python project structure. To the greatest degree possible the structure attempts to provide the basic infrastructure for a project. It is most certainly not adequate to the final needs of any given individual project. Rather, the project structure presented here attempts to be light-weight. The desire is to enable the Python developer to get a project up and running quickly, while minimizing the hindrance to allow for the inevitable organic growth

The project structure will be presented by way of narration. An example project will be constructed to explain and demonstrate. The example project is 'Hola Mundo'. 

.
├── bin
├── doc
├── hola_mundo
│   └── util
└── test
'Hola Mundo' will demonstrate support for all of the basic facilities listed above. The completed source from this narration can be cloned from GitHub. 
    git clone --branch the_start https://github.com/neoinsanity/hola_mundo.git
