A Rational

One of the most difficult issues facing the initial implementation of a project is to get a good start setting up the initial structure. A complete project should enable ease of development by allowing for quick iterations. In addition, a good project structure should aid in maintaining standards and conventions.

A basic project structure should provide the facilities for:

  • Source Control Configuration
  • Dev Environment Configuration
  • Package Control and Configuration
  • Test Configuration and Execution
  • Documentation Generation

While the project structure presented here is in now way definitive, it does provide for most typical projects. At the very least, it provides a project structure that can be extended as project requirements grow and shift under the pressure of evolving evolution.
