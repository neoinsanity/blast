Ah yes, the first blog. The one were the angst of a topic and stress of layout and presentation collide. Procrastination found ready allies to ensure some wasted hours. It can be amazing how much time can pass while flipping back and forth in edit and preview mode. Background change, font change, background color change, font color change, ... rinse and repeat.

The realization that presentation fiddling would be easier if there was an actual blog post with a good amount of content and content variety finally interrupted all the false starts. A spoonful of sugar helps the medicine got down, and for this blog that sugar was code syntax highlighting. While it would be simple enough to place code samples in an html block that would preserve the code format, it would be much more elegant to have actual code syntax highlighting.

To that end, a little web searching led to a fairly simple solution. The solution was to utilize SyntaxHighlighter, which is quite ubiquitous and easy to use. SyntaxHighlighter is a a javascript library that is self contained. It is open source and can be found at alexgorbatchev/syntaxhighlighter. There is plenty of material on how SyntaxHighlighter is used and installed in a variety of environments. For instructions on how to install SyntaxHighlighter for use on Blogger/Blogspt can be found here.

Is this meta?

SyntaxHighlighter has a wonderful array of languages that it supports. It is easy enough to lookup the full range of supported languages, Just take a step through the javascript, or take a look at the online documentation.

For the purposes of filling up a page to get this blog started, a couple of example uses of Syntaxhighlighters attribute flags will be demonstrated.

HTML/XML

The first example is how to get XML/HTML syntax highlighting. This is necessary to learn to get demonstrate the html syntax to highlighting examples in other languages. Here's the example of how to insert html sample code: 
      <html>
        <body>
          
Hola! Mundo

        </body>
      </html>
You will need to add the following highlight to your page. 
        <pre class="brush: xml">
          &lt;html&gt;
            &lt;body&gt;
              
Hola! Mundo

            &lt;/body&gt;
          &lt;/html&gt;
        </pre>
Keep in mind that to get the HTML, BODY, and PRE tags to show up in code examples. the use of the ampersand directive is necessary.

While it is simple enough to do plain text in a simple <block-quote>, SyntaxHighlighter gives nice line numbers. And the line numbers can be adjusted by setting a first-line property. 
        <pre class="brush: plain first-line: 42">
            Hola! Mundo
        </pre>
Which leads to: 
        Hola! Mundo

It is also possible to set a title for a code sample by utilizing the title directive. 
        <pre class="brush: bash title: 'Sample Bash'">
            #!/usr/bin/env bash
            echo "Hola Mundo"
        </pre>
Resulting in; 
        #!/usr/bin/env bash
        echo "Hola Mundo"

To highlight individual lines, which can be useful when there is a desire for emphasis. 
        <pre class="brush: js highlight: [2,5]">
        /*
        Hola Mundo Function
        */
        function HolaMundo(dude: "Mundo"){
          console.log('Hola! %s', dude);
        }
        </pre>
Producting; 
    /*
    Hola Mundo Function
    */
    function HolaMundo(dude: "Mundo"){
      console.log('Hola! %s', dude);
    }

In those cases where code samples may be large, it would be desirable to have the sample start off collapsed. 
        <pre class="brush: css collapse: true">
            #header {
                padding-top: 15px;
                border-radius: inherit;
            }
        </pre>
Leading to this collapse code sample. 
        #header {
            padding-top: 15px;
            border-radius: inherit;
        }

If the desire is to have a toolbar for the given code snippet, utilize the toolbar directive. 
        <pre class="brush: python toolbar: true">
        """A Class to greet the world"""
        class HolaMundo(object):

            def __init__(self, dude='Mundo'):
                self._dude = dude

            def grito(self):
                print 'Hola!', self.dude
        </pre>
Which removes the toolbar as shown below; 
    """A Class to greet the world"""
    class HolaMundo(object):

        def __init__(self, dude='Mundo'):
            self._dude = dude

        def grito(self):
            print 'Hola!', self.dude

Sometime the use of gutters is not desired for a code sample. These can be disabled utilizing the gutter directive. 
        <pre class="brush: sql gutter: false">
            select * 
            from Mundo 
            where name = "Mundo";
        </pre>
    
        select * 
        from Mundo 
        where name = "Mundo";

All of these directive can be mixed and matched.
